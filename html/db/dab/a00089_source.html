5 ## Macros that affect how MACE works
6 
7 | *Macro* | *Effect* |
8 |--------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
9 | MACE_DEBUG | Does more error checking and more verbose error messages. Automatically defined in Debug configurations. If defined to be 0 before including any MACE headers, will never be defined. |
10 | MACE_EXPOSE_WINAPI | If defined, classes that use Winapi will have functions to allow for direct access to any Winapi variables. Will not work if MACE_WINAPI is not defined. |
11 | MACE_EXPOSE_POSIX | If defined, classes that use POSIX will have functions to allow for direct access to any POSIX variables. Will not work if MACE_POSIX is not defined |
12 | MACE_EXPOSE_OPENGL | If defined, the OpenGL abstraction layer and renderer will be exposed by including MACE.h |
13 | MACE_EXPOSE_GLFW | If defined, os::WindowModule will expose the underlying GLFWwindow |
14 | MACE_EXPOSE_ALL | If defined, defined all macros starting with MACE_EXPOSE_* |

25 #### OpenCV
26 
27 MACE has optional interoptibility with OpenCV. To link OpenCV with CMake, set the OpenCV_* cache variables when running CMake.
28 
29 MACE will attempt to automatically detect OpenCV and defined MACE_OPENCV if found.
30 
31 To manually link OpenCV yourself, define MACE_OPENCV to be 1 before including any MACE header file:
32 
33 ```
34 //specify that OpenCV is linked
35 #define MACE_OPENCV 1
36 #include <MACE/MACE.h.
37 ```
38 
39 MACE will include `<opencv2/opencv.hpp>` if MACE_OPENCV is defined.
40 
41 Additionally, if CV_VERSION is defined, MACE_OPENCV will also be defined automatically.
42 
43 To request to never use OpenCV, even if it is included, define MACE_OPENCV to be 0 or false.
44 
45 ```
46 //both of the following lines tell MACE to never use OpenCV
47 #define MACE_OPENCV 0
48 #define MACE_OPENCV false
49 #include <MACE/MACE.h>
50 ```

60 ### Compilers
61 
62 The macro MACE_COMPILER has the compiler ID. The MACE_COMPILER_VERSION macro has the compiler version.
63 
64 If Microsoft Visual Studio is used, MACE_MSVC is defined to be the Visual Studio version number. Additionally, if the IDE is being used, MACE_MSVC_IDE is defined to 1.
65 If MSCV CL is being used, MACE_MSVC_CL is defined to 1. If MSVC 2005 compiler is used, MACE_MSVC_2005 is defined to 1.
66 
67 Otherwise, one of the following macros will be defined to the compiler version:
68 
69 * MACE_BORLAND
70 * MACE_MINGW
71 * MACE_CYGWIN
72 * MACE_CLANG
73 * MACE_GNU
74 * MACE_HP
75 * MACE_INTEL
76 * MACE_ORACLE
77 * MACE_TEXAS_INSTRUMENTS
78 * MACE_WATCOM
79 * MACE_ADSP
80 * MACE_CRAY
81 * MACE_MIPSPRO
82 * MACE_PORTLAND
83 * MACE_PATHSCALE

88 ### Informational macros
89 
90 | *Macro* | *Meaning* |
91 |-----------------------------|----------------------------------------------------------------------------------------------------------------|
92 | MACE_INCLUDED | Always defined to be 1 when MACE is included in a project |
93 | MACE_TESTS | Defined if CMake built the testing suite. |
94 | MACE_DEMOS | Defined if CMake built the demos |
95 | MACE_INCLUDES | Defined to be the directory in which the include files are located. This macro is not stringified. |
96 | MACE_DIRECTORY | Defined to be the directory where the MACE source came from. This macro is not stringified. |
97 | MACE_SYSTEM_NAME | Defined to be the name of the operating system in which MACE was configured on. This macro is not stringified. |
98 | MACE_SYSTEM_VERSION | The version of the operating system in which MACE was configured on. This macro is not stringified. |
99 | MACE_PROCESSOR_NAME | The name of the processor on which MACE was configured on. This macro is not stringified. |
100 | MACE_POINTER_SIZE | Size of a void* on this system. |
101 | MACE_32_BIT | Whether this system is 32 bit. Defined if MACE_POINTER_SIZE == 4. |
102 | MACE_64_BIT | Whether this system is 64 bit. Defined if MACE_POINTER_SIZE == 8. |
103 | MACE_DYNAMIC_LIBRARY_PREFIX | The prefix for dynamic libraries (such as lib.) Some systems may not have a prefix. |
104 | MACE_DYNAMIC_LIBRARY_SUFFIX | The suffix for dynamic libraries (such as .so, .dll, or .a.) |
105 | MACE_LITTLE_ENDIAN | Whether the system uses little endian (significant bit/byte last) |
106 | MACE_BIG_ENDIAN | Whether the system uses big endian (significant bit/byte first) |

109 ### Utility macros
110 
111 | *Macro* | *Meaning* |
112 |---------------------------------|---------------------------------------------------------------------------------------------------------|
113 | MACE_FUNCTION_EXPORT | Modifier that tells the compiler that this function should be exported to the global symbol table. Could be empty |
114 | MACE_FUNCTION_IMPORT | Modifier that tells the compiler that this function will be imported from an exported symbol. Could be empty. |
115 | MACE_HAS_ATTRIBUTE(attr) | If not defined previously, returns 0 if specified attribute is supported, 1 otherwise. If attributes are not supported by the compiler, always returns 0 |
116 | MACE_FALLTHROUGH | If not defined previously, if the compiler supports the fallthrough attribute, MACE_FALLTHROUGH is defined to that. Otherwise, defined to nothing |
117 | MACE_HAS_INCLUDE(incl) | If not defined previously, returns 1 if the system has the specified include file, 0 otherwise. Not all compilers support this. If unsupported by the compiler, returns 0 always. |
118 | MACE_CONSTEXPR | If not defined previously, defined to be `constexpr` on compilers that `constexpr` is supported, otherwise becomes nothing |
119 | MACE_STATIC_ASSERT(condition, message) | If not defined, defined to be `static_assert`. Can be defined before including MACE headers to change the static assert used by MACE |
120 | MACE_STRINGIFY(name) | Directly stringifies a macro |
121 | MACE_STRINGIFY_NAME(name) | Stringifies a macro name. If the expanded macro is empty, this function returns "" |
122 | MACE_STRINGIFY_DEFINITION(name) | Stringifies the definition of a macro. If the macro doesn't have a definition, this function returns "" |
