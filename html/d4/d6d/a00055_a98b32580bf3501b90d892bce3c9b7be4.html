void mc::gfx::ogl::VertexBuffer::setAttributePointer ( const Byte  attribSize,
const Enum  type,
const bool  normalized = 0,
const Index  stride = 0,
const void *  pointer = nullptr 
)

Tells OpenGL how to interpret array data.

Must be called in order for rendering to work. Must be used in conjuction with Buffer::setData(const ptrdiff_t, const GLvoid*, const Enum)

Behind the scenes, it will call either glVertexAttribPointer or glVertexAttribIPointer based on the arguments provided.

Parameters
attribSizeThe size of the component. Must be 1,2,3,4 or GL_BGRA (which is 4.)
typeThe data type of the component, like GL_FLOAT or GL_UNSIGNED_INT
normalizedWhether the data should be normalized to a float or keep the initial value. False by default.
strideThe offset between components. 0 means every component is tightly packed. 0 by default.
pointerAn offset of the first component. 0 by default.
See also
https://www.opengl.org/wiki/GLAPI/glVertexAttribPointer
VertexArray
Attention
This uses an OpenGL function and must be called in a thread with an OpenGL context. Otherwise, an error will be thrown.
Exceptions
GL_INVALID_OPERATIONIf the current thread does not have an OpenGL context
