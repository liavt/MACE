void* mc::gfx::ogl::Buffer::mapRange ( const Index  offset,
const Size  length,
const unsigned int  access 
)
inherited

Maps a range of data in this Buffer to a pointer on the CPU side.

Parameters
offsetThe starting offset to map
lengthHow much data to map
accessA combination of access flags to determine how the data will be used. These flags determine how fast or slow the mapping will take.
See also
https://www.opengl.org/wiki/GLAPI/glMapBufferRange
Buffer::map(const Enum)
Warning
DO NOT DELETE THE POINTER RETURNED. IT IS NOT DYNAMICALLY ALLOCATED.
After Buffer::unmap() is called, the pointer returned may be unsafe. Using it will be undefined.
Attention
When using this function don't forget to use Buffer::unmap() when you are done using the pointer
This uses an OpenGL function and must be called in a thread with an OpenGL context. Otherwise, an error will be thrown.
Exceptions
GL_INVALID_OPERATIONIf the current thread does not have an OpenGL context
