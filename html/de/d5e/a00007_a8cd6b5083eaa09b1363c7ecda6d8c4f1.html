void* mc::gfx::Buffer::map ( const Enum &  access = GL_READ_WRITE)
inherited

Maps the data in this Buffer to a pointer on the CPU side.

May be slow.

Parameters
accessWhich access to use. Must be GL_READ_ONLY, GL_WRITE_ONLY, or GL_READ_WRITE.
See also
https://www.opengl.org/wiki/GLAPI/glMapBuffer
Buffer::mapRange(const Index&, const Size&, const unsigned int&)
Warning
DO NOT DELETE THE POINTER RETURNED. IT IS NOT DYNAMICALLY ALLOCATED.
After Buffer::unmap() is called, the pointer returned may be unsafe. Using it will be undefined.
Attention
When using this function don't forget to use Buffer::unmap() when you are done using the pointer
This uses an OpenGL function and must be called in a thread with an OpenGL context. Otherwise, an error will be thrown.
Exceptions
GL_INVALID_OPERATIONIf the current thread does not have an OpenGL context
